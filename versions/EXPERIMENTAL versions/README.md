## What does EXPERIMENTAL mean here?

In `muri_ai`, **EXPERIMENTAL versions are not dangerous or unusable**.

They are versions that:
- focus more on **testing new ideas** than on fixing every bug
- may contain unfinished or imperfect logic
- aim to be **functional at a basic level**
- keep syntax and usage **very similar to BETA and STABLE versions**

In practice, these versions are still **beginner-friendly**.

---

## How EXPERIMENTAL versions are used

EXPERIMENTAL versions are used to:
- try new training logic
- test alternative response systems
- experiment with internal structure
- explore ideas before stabilising them

Bug fixing is **not the main priority**, but basic functionality is always intended.
